# Orbitherm Studio

FreeCAD-based thermal modeling workbench for spacecraft and engineering systems.

FreeCADベースの宇宙機・工学システム向け熱解析モデリング環境

---

## 概要

**Orbitherm Studio** は、FreeCAD 上で熱ネットワークモデルを構築するワークベンチです。
ノード・コンダクタ・放射モデルの作成から、軌道熱入力の計算、外部ソルバーへのエクスポートまでを統合した環境を提供します。

FreeCAD の Workbench 一覧には **Orbitherm** として表示されます。

> ソルバー側は **Orbitherm Solver**（別リポジトリ）が担当します。
> Orbitherm Solver は SINDA ライクな熱ネットワークソルバーで、定常解析・過渡解析に対応します。

---

## 構成

- **modeling** — 熱モデル構築・コンダクタンス計算
  - モデル準備、面プロパティ編集、熱容量・伝熱・輻射コンダクタンス計算、エクスポート、可視化
- **orbit_heat** — 軌道熱計算
  - 軌道計算と描画、熱入力 CSV、輻射モデルへの軌道熱適用
- **bridge** — 外部ソルバー連携・ファイル出力
- **post** — ポスト処理・結果可視化
- **solver** — ソルバー連携（Orbitherm Solver との接続領域）

---

## インストール

1. この `orbitherm-studio` フォルダを FreeCAD の Mod ディレクトリに配置します。
   - ユーザー Mod: `%APPDATA%\FreeCAD\Mod\orbitherm-studio`（Windows）
   - または FreeCAD インストール先の `Mod/orbitherm-studio`
2. FreeCAD を起動し、ワークベンチ一覧から **Orbitherm** を選択します。

## コマンド一覧

### Modeling メニュー / ツールバー

#### モデル準備・形状簡略化

| コマンド | 説明 |
|---|---|
| **モデルを準備** | 選択したオブジェクトの面ごとにメッシュを生成し、面中心ノードを作成します。オブジェクト選択で全面、面をサブ選択すると選択面のみ変換します。Shell などサーフェースのみのオブジェクトも対応しています。 |
| **形状の簡略化（穴・フィレット削除）** | 選択した Part に対して、指定直径以下の穴・指定 R 以下のフィレットを削除した新しい形状を作成します。解析用モデルの単純化に使用します。 |
| **形状の簡略化（選択した面のみ）** | オブジェクトを選択した状態で削除したい面を Ctrl+クリックでサブ選択し、選択した面のみを Defeaturing で削除した新しい形状を作成します。 |

#### プロパティ・マテリアル

| コマンド | 説明 |
|---|---|
| **プロパティ編集** | 選択した面メッシュオブジェクトの熱物性・光学特性（太陽光吸収率・赤外放射率・透過率・アクティブ面など）を個別に編集します。面メッシュ（`Face_*`）を選択した状態で有効になります。 |
| **面の一括プロパティ設定** | 輻射モデル内の全面に同じ熱物性・光学特性を一括設定します。 |
| **マテリアル編集** | マテリアルライブラリ（`materials.json`）を GUI で編集・追加・削除します。 |

#### コンダクタンス計算

| コマンド | 説明 |
|---|---|
| **熱容量計算** | 各ノードの面積・厚さ・密度・比熱から熱容量 (W·s/K) を計算し、ノードの `ThermalMass` に書き込みます。 |
| **伝熱コンダクタンス計算** | 隣接メッシュ間の共有辺長・厚さ・熱伝導率から伝熱コンダクタンス (W/K) を計算します。 |
| **輻射コンダクタンス計算** | モンテカルロ法レイトレーシングでビューファクターを計算し、輻射コンダクタンス係数（R' = ε × Vf × A）を算出します。宇宙空間への輻射は `SPACE.9999` ノードとして出力されます。射線数はダイアログで指定できます。 |
| **伝熱コンダクタンスを追加** | 選択した 2 ノード間に手動で伝熱コンダクタンスリンクを追加します。ノード（`Node_*`）を 2 つ選択した状態で有効になります。 |

#### サーフェース操作

| コマンド | 説明 |
|---|---|
| **サーフェースを分割** | 選択した FaceGroup のメッシュをグリッド（U × V）で分割し、サブサーフェースとノードを作成します。 |

#### 表示

| コマンド | 説明 |
|---|---|
| **表示設定** | アクティブ面・太陽光吸収率・赤外放射率・透過率・デフォルトの各表示モードを排他的に切り替えるダイアログです。 |
| **表示・伝導パラメータ設定** | ノードラベル表示・コンダクタンス線・伝導コンダクタンスのペア選定に使用するパラメータを設定します。設定は FreeCAD ユーザー設定に保存されます。 |
| **アクティブ面 表示** | アクティブ面の設定に応じて面を色分け表示します。 |
| **吸収率 表示** | 太陽光吸収率をコンターカラーで表示します。 |
| **放射率 表示** | 赤外放射率をコンターカラーで表示します。 |
| **透過率 表示** | 透過率をコンターカラーで表示します。 |
| **表示をデフォルトに戻す** | アクティブ面・吸収率・放射率・透過率の色分けをやめ、全面をデフォルト色に戻します。 |

#### エクスポート

| コマンド | 説明 |
|---|---|
| **Thermal Model Export** | HEADER / OPTIONS / CONTROL を設定ダイアログで指定し、ノード・伝熱・輻射コンダクタンスを熱解析ソルバー用 `.inp` ファイルとして出力します。 |

### Orbit Heat メニュー / ツールバー

| コマンド | 説明 |
|---|---|
| **軌道計算と描画** | 軌道要素（TLE またはパラメータ入力）・熱環境（太陽定数・アルベド・地球赤外）・姿勢モードを設定し、面ごとの熱入力を計算して FreeCAD 3D ビューに軌道を描画します。計算結果は次のコマンドで CSV 保存できます。 |
| **熱入力 CSV を保存** | 「軌道計算と描画」で計算済みの熱入力データを CSV ファイルに保存します。 |
| **計算して CSV 保存** | 軌道・環境を設定し、熱入力を計算してそのまま CSV のみ保存します（3D 描画は行いません）。 |
| **軌道コマ送り** | 各計算点での衛星の位置・姿勢をステップごとに確認するダイアログです。 |
| **輻射モデルに軌道熱を適用** | 最後に計算した軌道・熱環境で面ごとの熱入力を算出し、輻射モデル各ノードの `HeatSource` に軌道周回平均値を書き込みます。面ごとの熱入力は CSV に保存することもできます。 |
| **Clear Orbit Visualization** | FreeCAD ドキュメント内の `OrbitVisualization` グループとその子オブジェクトをすべて削除します。 |

---

### Post メニュー

| コマンド | 説明 |
|---|---|
| **Post Processing** | 熱解析ソルバーが出力した `.out` ファイルを読み込み、ノード温度をモデル上にコンター表示します。過渡解析では複数時刻の切り替えと表示温度範囲の指定が可能です。 |

## 標準的なワークフロー

```
1. モデルを準備          ← オブジェクトを選択して実行（面メッシュ・ノードを生成）
2. プロパティ編集         ← 各面の材料・光学特性を設定
3. 熱容量計算            ← ノードの熱容量を算出
4. 伝熱コンダクタンス計算  ← 隣接面間の伝熱コンダクタンスを算出
5. 輻射コンダクタンス計算  ← ビューファクター・輻射コンダクタンスを算出
6. 軌道計算と描画         ← 熱入力を計算（必要に応じて CSV 保存）
7. 輻射モデルに軌道熱を適用 ← HeatSource をノードに書き込み
8. Thermal Model Export   ← ソルバー用 .inp を出力
9. （外部ソルバーで解析）
10. Post Processing        ← .out を読み込んで結果表示
```

## 移行状況

旧名称 `RadiationAnalysis` および `OrbitHeatWorkbench` は本ワークベンチに統合済みです。
コマンド登録名 `ThermalAnalysis_*` は内部互換性のため現在も使用されますが、
将来的に `Orbitherm_*` プレフィックスへ段階移行予定です。

旧 `RadiationAnalysis_*` コマンドへのエイリアスは互換維持のため引き続き登録されます。
